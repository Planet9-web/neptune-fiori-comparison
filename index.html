Starting Point
The Right Question to Ask

When EHP8 completes, Fiori becomes technically available at no additional licensing cost. That is a fair and important observation. This document addresses it directly.

The comparison is not Fiori licence cost vs. Neptune licence cost. That framing is incomplete. The real question is: what does it actually cost to solve the production confirmation problem — including development, implementation, maintenance, upgrade risk, and time to value — using each approach?

The Specific Problem

Operators are bypassing ZPP_CONF2 and reverting to CO11N. Labor and production actuals are not captured consistently. Supervisors spend approximately 1 FTE per site on variance correction every month. The experience needs to be simpler, mobile-ready, and consistent across global manufacturing sites with localisation requirements. This is a custom, operator-facing, mobile workflow — and that context matters when evaluating platform fit.

Summary

Custom Fiori development does not require EHP8 — it can begin today. EHP8 expands the range of standard Fiori apps available, but as the analysis below shows, no standard Fiori app meets the requirements of ZPP_CONF2. The real question is therefore what a custom Fiori build costs to deliver, maintain, and upgrade — compared to Neptune.

Why Neptune for This Use Case
Six Capabilities Directly Relevant to Pelican
01

No ABAP Rebuild Required

ZPP_CONF2 logic stays intact. Neptune exposes existing ABAP as REST APIs. Fiori requires rebuilding it as OData — costly and risky for proven business logic.

02

One Developer, End to End

Neptune's App Designer lets a single ABAP developer handle the full build. Standard Fiori requires three specialist roles: ABAP/OData, UI5 frontend, and SDK/deployment.

03

Offline and Mobile — Already Proven at Pelican

Neptune's offline-first mobile capability is already live at Pelican for Plant Maintenance. Advanced mobile features in Fiori — offline, native device integration, push notifications — require SAP Mobile Services or another mobile framework on top of the base licence.

04

ECC Compatible Today

Neptune installs into ECC as a certified ABAP add-on. Development starts now, independent of EHP8, and migrates cleanly to whatever SAP version Pelican lands on.

05

Predictable Cost at Scale

Neptune charges by named user. Everything included. BTP charges by API calls, memory, tenants, and services — costs increase as usage scales, making forecasting difficult.

06

Internal Team Can Own It

Neptune's standardised framework means Pelican's team maintains and changes the app without external consultants for routine updates. Custom Fiori apps require specialist upkeep indefinitely.
